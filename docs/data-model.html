BelegTool — Data Model Reference

The immutable, UI-agnostic data-driven core (core/). Generated from core/model.py, core/commands.py, core/engine.py, core/session.py. Self-contained — open offline.

Architecture. Everything here is pure data + pure functions: an immutable document, commands as data, an injected PDF engine for the impure byte work, and an editing session with undo/redo. No Tk, no rendering, no threads — fully unit-testable (tests/test_model.py, test_commands.py, test_engine_commands.py, test_split_merge.py, test_session.py, test_bridge.py).

entity Document — core/model.py

FieldTypeReq.Meaning
rootNode (folder)requiredThe root folder; the whole tree hangs off it.

Persisted shape (the data contract)

Document.to_dict() serialises structure only (no bytes). Per node: id, name, is_folder, status, position (= sibling index), vz_start, vz_end, pdf_length, is_compressed, dpi_original, dpi_current, no_compression, compression_no_gain, collapsed, compression_method, tags[], datev (root only), children[]. On disk a .belegtool is a single PDF carrying the pages plus this JSON under the PDF metadata key /JSONStructure (see docs/DATA_CONTRACT.md; bridge in core/bridge.py).

3 · Commands (the methods of data manipulation)

Commands are frozen data objects; the pure reducer apply(doc, cmd, engine=None) → Document interprets them. Invalid targets raise CommandError. core/commands.py

structural — no engine needed

CommandFieldsEffect / cascade
AddFolderparent_id, name, index?, new_id?Insert a new folder under a folder parent.
Renamenode_id, nameSet name (non-empty).
SetStatusnode_id, statusSet status (must be a valid status, incl. "" = no status). On a folder it cascades to every descendant document.
SetStatusManynode_ids, statusSet status on several nodes as one undo step; a folder in the selection cascades to its documents. Ids de-duped, missing ids skipped.
SetNoGainnode_idsFlag each existing leaf compression_no_gain=True (folders/missing ids skipped; idempotent). A derived verdict, not a human edit: the API applies it silently (DocumentSession.apply_silent → no undo/redo, no dirty, not in the log) when a compression evaluation finds nothing smaller, so the red "undecided" dot stays resolved across moves and reopen.
SetDatevprovenanceDATEV mode only (v3.10.0). Set (or clear with None) the datev provenance on the root node. In-process only (carries no bytes but never part of the JS wire command set, like SetNodeBytes) and applied silently (apply_silent → no undo/dirty) — it is origin metadata, not a user edit — yet it changes the document so a later save persists it. Set on open from a checkout path / on file-to-DATEV; cleared by Save As (breaks the link).
SetCollapsednode_id, collapsedFolder only → set collapsed (rejects leaves).
SetAllCollapsedcollapsedSet collapsed on every folder at once (one undo step).
SetPeriodnode_id, vz_start, vz_endSet the accounting period.
SetTagsnode_id, tagsReplace a node's whole tags set (folders & leaves); normalised: trimmed, empties dropped, de-duplicated first-seen.
TagManynode_ids, tag, addAdd (add=True) or remove ONE tag across several nodes as one undo step, keeping each node's other tags. Ids de-duped, missing ids skipped, empty tag a no-op; adding an already-present tag (or removing an absent one) leaves that node unchanged. Powers multi-select tagging.
Deletenode_idRemove the node + its whole subtree (not the root).
DeleteManynode_idsRemove several nodes + their subtrees as one undo step; nested members de-duplicated (a node already inside a selected ancestor is skipped).
Movenode_id, new_parent_id, index?Relocate a node + subtree under a folder; not into its own subtree, not the root.
MoveManynode_ids, new_parent_id, index?Move several nodes (any depth) under one folder, order-preserving; nested members de-duplicated.
GroupIntoFoldernode_ids, parent_id, name, index?Create a folder and move all selected nodes into it (the non-concatenating counterpart to Merge).
InsertNodesparent_id, nodes, index?Insert already-built nodes (freshly imported files) under a folder. In-process only (carries bytes); not part of the wire/JSON command set.
SetNodeBytesnode_id, dataReplace a leaf's content with new PDF bytes — the PDF-Tool save-back (form-fill / typewriter text baked by PDF.js). The new bytes become the node's source and compression state resets (mirrors Rotate). In-process only (carries bytes); not part of the wire/JSON command set. Dispatched by save_node_back (bound node) and, for a directly-opened .pdf bridge session, by save_pdf_bytes (session+disk) / update_pdf_bytes (session-only bake before a guarded DATEV write-back).

engine-backed — require an injected Engine

CommandFieldsEffect / cascade
Compressnode_id, dpi=150, method=Noneleaf → set current_data, is_compressed, dpi_current, compression_method; no-op if no gain; rejects no_compression.
Commitnode_idPromote current_data → original_data; clear compression flags.
Resetnode_idLeaf only — drop the compressed variant (clear current_data + flags); rejects folders and a committed node without a source.
Rotatenode_id, directionRotate the original (right/left/180); invalidates any compressed variant.
Splitnode_idLeaf → one page-leaf per page, in place (single page = no-op). Parts inherit the original's status.
SplitIntonode_id, size=1, into_folder=FalseLeaf → chunks of size pages (1 = per page). into_folder wraps the parts in a NEW folder named after the node; else in place. In-place single-chunk = no-op.
Mergenode_ids≥2 sibling leaves → one leaf. DPI-conflict invariant: same DPI keeps the merged compression; differing/mixed/no_compression drops it and sets no_compression. Status: all same → kept; any difference → no status.

state DocumentSession — core/session.py

MemberPurpose

Replay invariant (tested): apply_all(session.initial, session.log, engine) == session.document — holds through undo/redo/divergence, so a session is fully reconstructable / persistable from its command log.
