# MSIX packaging for the Microsoft Store

Build a Store-ready MSIX of BelegTool so Microsoft signs it (eliminates the Norton
CyberCapture / SmartScreen false positives on the unsigned `.zip`) and enables auto-updates.

| File | Purpose |
|---|---|
| `AppxManifest.xml` | The package manifest (identity, capabilities, tiles, `.belegtool` association). |
| `build_msix.ps1` | Build app → lay out → pack `.msix` (and optionally a self-signed test copy). |
| `generate-assets.ps1` | Render the tile/logo PNGs from `assets/icon.ico`. |
| `Assets/`, `msix_layout/`, `out/` | Generated — git-ignored. |

## Store identity (permanent — must match Partner Center)

```
Name:           TobiasHeinrich.BelegTool
Publisher:      CN=2A87E267-9C96-4414-B440-06EB61A80222
PublisherName:  Tobias Heinrich
PFN:            TobiasHeinrich.BelegTool_0pmf53364w2ma
Store ID:       9PL4D25N00XD
```
Version in the manifest is 4-part with revision 0 (`X.Y.Z.0`); `build_msix.ps1` syncs
`X.Y.Z` from `version_info.py`.

## Build

```powershell
# Store-upload package (UNSIGNED — Microsoft re-signs on ingestion):
.\packaging\build_msix.ps1
#   -> packaging\out\BelegTool-<ver>-x64.msix      <- upload this to Partner Center

# Also produce a self-signed copy for LOCAL sideload testing:
.\packaging\build_msix.ps1 -SkipBuild -Sign
```
Requires the **Windows 10/11 SDK** (`makeappx.exe`, `signtool.exe`) — the script locates the
latest x64 copy under `…\Windows Kits\10\bin`.

## Required Store assets (PNG, generated by `generate-assets.ps1`)

| File | Size (px) | Use |
|---|---|---|
| `Square44x44Logo.png` | 44×44 | app list / taskbar / file-type icon |
| `Square150x150Logo.png` | 150×150 | medium tile (**required**) |
| `Square71x71Logo.png` | 71×71 | small tile |
| `Square310x310Logo.png` | 310×310 | large tile |
| `Wide310x150Logo.png` | 310×150 | wide tile |
| `StoreLogo.png` | 50×50 | Partner Center / Store listing |

These are auto-rendered from `assets/icon.ico` (good enough for a first submission; replace
with hand-crafted artwork — and per-scale variants `*.scale-200.png` etc. — for crisp large
tiles). The **Store listing** also needs ≥1 **screenshot** (1366×768 or 7.68"×4.32"), added
in Partner Center (not in the package).

## Certification note — broadFileSystemAccess justification

Paste into Partner Center → submission → **"Notes for certification"**:

> BelegTool is a professional desktop tool for organising PDFs and bookkeeping receipts
> ("Belege"). It requires **broadFileSystemAccess** because the user opens and saves
> `.belegtool` documents and source files (PDFs, scans, images, Office files, e-mails) from
> **arbitrary, user-chosen locations** — including company **network shares** and external
> drives — and the app maintains a persistent working document across many files in a
> folder tree. The standard file-picker / brokered file APIs are insufficient: the app
> performs batch tree operations (import, merge, split, compress, export) and an optional
> single-writer lock over a shared network path, which require direct read/write access to
> locations the user selects, not one-file-at-a-time picker access. No files are accessed
> without the user choosing them. The capability is **not** combined with the
> Documents/Pictures/Videos library capabilities.

## Local sideload test (before submitting)

1. Build the signed test package:
   ```powershell
   .\packaging\build_msix.ps1 -SkipBuild -Sign
   ```
2. Trust the self-signed publisher cert **once** (elevated PowerShell):
   ```powershell
   Import-Certificate -FilePath .\packaging\out\BelegTool-test-publisher.cer -CertStoreLocation Cert:\LocalMachine\TrustedPeople
   ```
3. Install the test package:
   ```powershell
   Add-AppxPackage .\packaging\out\BelegTool-<ver>-x64-TEST-signed.msix
   ```
4. Launch from the Start menu; verify the UI renders, open/save/export work, and the
   `.belegtool` association opens files. Remove with:
   ```powershell
   Get-AppxPackage *TobiasHeinrich.BelegTool* | Remove-AppxPackage
   ```
The self-signed cert + signature are **local-test only** — the package uploaded to the Store
is the **unsigned** `BelegTool-<ver>-x64.msix`; Microsoft signs it.

## WebView2 runtime (important)

There is **no MSIX framework dependency** to declare for the Edge WebView2 Runtime — it's a
system component, not a Store framework package. So:
- **Win11** ships it in-box (works out of the box).
- **`host.py`** checks at startup and shows a message + download link instead of a blank
  window if it's missing (`BELEG_SKIP_WEBVIEW2_CHECK=1` bypasses).
- For guaranteed coverage on minimal/Win10 machines, **bundle a fixed-version WebView2
  runtime** in the package and point pywebview at it — an optional later enhancement.

## Notes / gotchas
- Full-trust PyInstaller app → `EntryPoint="Windows.FullTrustApplication"` + `Executable=`
  the onedir exe. No `desktop6`/`desktop2` extensions needed.
- `PackageIntegrity` content enforcement is on → requires Windows **2004 (19041)**; that's the
  manifest `MinVersion`. (Drop PackageIntegrity if you must support 1903/1909 — both EOL.)
- MSIX virtualises install/registry; the app needs no installer/uninstaller logic.
