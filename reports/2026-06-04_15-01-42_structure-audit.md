## What's well-structured (keep)

- **Macro logic/UI separation holds.** `core/` and `services/` have **zero** UI imports (no pywebview/tkinter) — verified. The frontend is fully isolated in `webui/`.
- **`core/` is a genuine data-driven domain** — immutable model, pure reducer with a handler registry, injected engine; headless-testable. The reference layer.
- **`services/` is a clean stateful-infra package** (render cache, CPU fairness, variant store) with rendering/CPU injected.
- **Frontend logic is UI-free and unit-tested** (`selection.js`, `treeNav.js`, `core.js` + their `.test`).
- **Required docs present**: `docs/data-model.html`, `DATA_CONTRACT.md`, `manual_tests/`, `FEATURES_REQUIRED.md`.
- **Tests are co-located by concept and comprehensive** (322 cases).

## Structural findings

| # | Severity | Finding | Suggestion |
|---|---|---|---|
| 1 | **Medium** | **The repo root is a flat dump that mixes three layers.** Alongside the entry point (`host.py`) and ports (`tasks`, `log_config`, `tools`) sit **6 heavyweight domain/IO modules** (`pdf_node`, `pdf_storage`, `toc_export`, `compress_pdf_bytes`, `universal_importer`). `core/` depends *out* to them via bare top-level imports (`from pdf_storage import …`, `from toc_export import …`, `from universal_importer import …`). So the clean `core/` sits on an **unpackaged** IO/conversion layer — the dependency exists but the tree doesn't express it, hurting discoverability and inviting accidental new top-level coupling. | Package the IO/conversion modules into a named layer, e.g. `formats/` (or `io/`): `pdf_node`, `pdf_storage`, `toc_export`, `compress_pdf_bytes`, `universal_importer`. Move ports into `infra/` (`tasks`, `log_config`, `tools`). Then `core` → `formats`/`infra` is an explicit, readable dependency. **High churn** (every importer, the PyInstaller spec, tests) → do it as one deliberate pass, not piecemeal. |
| 2 | Low–Med | **`universal_importer.py` (685) is a multi-responsibility god-module** — type detection + signature/magic checks + image/HEIC/Office(COM)/HTML/text conversion + zip/tar/email extraction in one class+module. | Split into an `importers/` subpackage: `detect.py` (signatures/magic), `image.py`, `office.py` (COM), `web.py` (html/text), `archive.py` (zip/tar), `email.py`. Keep a thin `UniversalImporter.convert` dispatcher. |
| 3 | Low | **`core/` mixes the pure domain with the IPC transport.** `pipe`/`protocol`/`server`/`client`/`cli`/`__main__` (win32 named-pipe + CLI) live next to `model`/`commands`/`engine`. Platform-specific transport sitting beside the pure model muddies the "core = pure logic" story. | Move the transport into `core/ipc/` (or a top-level `transport/`). The domain stays import-clean; the CLI entry points group together. |
| 4 | Low | **Dead Tk-era methods in `pdf_storage.py`** — `perform_move`, `get_full_gui_plan`, `_move_nodes_to_parent`, `_get_clean_selection`, `get_structure_json`, `extract_pages` return a "GUI Umbauplan"/utilities the React app never uses; only `test_pdf_storage_additional.py` pins them. ~70 lines of dead surface in a load-bearing file. | Remove the methods + their tests (tree moves go through `core/commands.Move`). Trims the carrier to its real I/O role. |
| 5 | Low | **Vestigial root `__init__.py` (0 bytes).** Makes the repo root a package that nothing imports; can confuse tooling/packaging and blurs "what is the top-level namespace." | Verify pytest import-mode/PyInstaller don't rely on it, then remove. |
| 6 | Low | **`webui/src` mixes pure logic with components.** `core.js` (bridge), `selection.js`, `treeNav.js` are UI-free logic sitting at the same level as `App.jsx`/`Tree.jsx`/… | Group them under `src/lib/` (or `src/logic/`) to mirror, on the frontend, the logic/UI split the workspace mandates. Low effort, improves readability. |
| 7 | Info | **`App.jsx` (653) is a large container** holding most app state + handlers. Known; not urgent. | Decompose opportunistically (e.g., extract the keyboard/grab logic and the preview-request wiring into hooks) as it grows. |

## Summary

- **Counts:** 0 Critical · 0 High · 1 Medium · 5 Low · 1 Info.
- **Health:** The *architecture* is sound — the data-driven domain, the infra split, and the logic/UI boundary are all real and verified. The weakness is **packaging hygiene at the repo root**: a strong `core/`+`services/` coexists with a flat pile of unpackaged IO/conversion modules it depends on, plus a god-module importer and some dead Tk-era surface.

### Sequencing (cheap → expensive)
1. **Now (cheap, low-risk):** remove the dead `pdf_storage` Tk-era methods + tests (finding 4); investigate/remove the root `__init__.py` (finding 5).
2. **Soon (moderate):** split `universal_importer` into `importers/` (finding 2); move frontend logic to `src/lib/` (finding 6).
3. **Deliberate pass (high churn):** introduce `formats/` + `infra/` packages and (optionally) `core/ipc/` (findings 1, 3) — one focused refactor with the full suite as the net, updating the PyInstaller spec and all imports together.
