## Findings

| Category | Severity | Finding | Location | Suggestion |
|---|---|---|---|---|
| Security / DoS | **High · NEW** | **Variant-blob ZIP bomb on open.** `variant_blobs.unpack()` decompresses embedded `variant_*` ZIP attachments with **no caps** (per-entry / total / count) — reached **automatically on `open()` of any `.belegtool`** via `seed_variants_from_file`. A malicious `.belegtool` (just a PDF + a deflate-bomb attachment whose uid matches a `/JSONStructure` node) expands to tens of GB → OOM crash on open. The sibling readers `extract_zip/tar_to_structure` were hardened (`3cef204`); this one was missed. | `services/variant_blobs.py:30-39`; `core/api.py:316-320`→`services/variant_store.py:127` | Read each entry via `z.open(name).read(CAP+1)`; enforce per-entry + running-total + entry-count caps + an attachment-count cap in `seed_variants_from_file` (reuse `archives.py` constants). |
| Lifecycle / correctness | **Medium · NEW** | **Stale cancel-token permanently disables compression** after delete→undo (and after deleting a shared-uid node in one of two windows on the same file). `dispatch()` sets the node's cancel `Event`; nothing clears it on revival → `compress_options` returns `[]` forever, the "undecided" dot never resolves. **Reproduced.** | `core/api.py:118-127` (`_cancel_token`), `382-388` (`dispatch`) | Clear/replace tokens of ids present in the new doc on dispatch/undo/redo; give `dispatch` the same "still live elsewhere" guard `close_session` has. + regression test. |
| Concurrency | **Medium · NEW** | `RealEngine._persisted` written by `seed_variants` (during `open()`, **outside** `CoreApi._lock`) while `variants_for`/`evaluated` iterate it under the lock on other pywebview threads → `RuntimeError: dictionary changed size during iteration` can fail an open/dispatch. | `core/engine.py:59,71-73,85,90-93` | Guard `_persisted` with `_mcache_lock`; snapshot `items()`. |
| Concurrency | **Medium · NEW** | `_pcount` read/written **without the lock** in `_count_for` (render-seed worker + js threads) while `close_session` iterates it under the lock → a concurrent insert raises inside `close_session`, swallowed by `host._bind_close`'s bare `except` → **silent whole-session leak** (bytes + undo log). Defense-in-depth only (the `%PDF`/parse checks still catch EXE-as-PDF), but the explicit EXE-refusal does nothing on the common route. | `universal_importer/importer.py:158` vs `177-206` | Call the verifier on the file head in the path branch too. |

## Confirmed correct (both agents)
- **All `dbc9f5a` fixes** (permanent aria-live regions, `_touch_view_dir`, drag-ghost `t()`, `aria-expanded`, shared-uid `close_session` test) **and the prior round** (`close_session` lifecycle, `localizeMessage`, `mergeableIds`, committed-node merge/split/rotate invariants vs drop-source-on-save) — verified, no regressions.
- Security still-solid: no eval/pickle/exec/shell on any path; ZIP/TAR **structure** readers' caps intact; no Zip-Slip / archive recursion / email-attachment traversal; xhtml2pdf SSRF/LFI/XXE blocked; JS bridge exposes no unguarded file-write/`save(path)`/`release`/`close_session`, and untrusted PDFs are server-rasterized to PNG (no live JS in the WebView); logging opt-in, no PII.

## Summary
- **Counts:** Critical 0 · **High 1** · **Medium 4** · Low ~10 · Info several. (vs the prior report's 0/0/0 — the focused passes dug deeper.)
- **Top 3:** (1) **variant-blob ZIP bomb** — cheap, high-value, crash-on-open from a hostile `.belegtool`; (2) **stale cancel-token** permanently kills compression after delete→undo (reproduced); (3) the two **unlocked-dict concurrency races** (`_persisted`, `_pcount`) that can fail an open/dispatch or silently leak a session on close.
- **Release call:** the **High (variant bomb)** is a cheap fix and worth doing before the next release (embarrassing crash-on-open). The 3 Mediums (cancel-token + 2 concurrency races) are real and should follow. The Lows are backlog.
