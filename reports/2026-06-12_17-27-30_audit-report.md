## Confirmed correct (8 fixes + 12 verifications тАФ nothing broken)
- **Variant attachment bomb cap (the reworked High):** `read_raw_bytes()` is the *raw/compressed* stream and the size pre-check runs **before** any decode; the single `zlib.decompressobj().decompress(raw, cap+1)` hard-bounds output in one C-level call (leftover `unconsumed_tail` тЖТ skip, decompressobj is local/discarded so a bomb can't continue later); **double-Flate / non-Flate / any `/DecodeParms` are refused (never decoded blind)**; per-attachment + running-total caps both enforced without corrupting `data_by_id`/doc; legit `ZIP_STORED` blobs reconstruct **byte-identical** (proven by a real `CoreApi.save`+pikepdf round-trip test). Skip is fully consistent downstream (no `KeyError`/`None` deref; node just recomputes).
- **`_dcache` byte-budget eviction has NO stale-digest risk:** eviction (`popitem(last=False)`) removes key+value together under the lock, so no key outlives its bytes' strong ref тЖТ a reused `id()` can't hit a stale entry; and the lookup independently re-checks `ent[0] is pdf_bytes` before trusting the digest. Byte accounting symmetric under the lock.
- **Cancel-token revive (failure path + open/import) introduces no race:** the live-elsewhere guard makes the feared cross-op revive structurally impossible; revive only deletes the dict entry тАФ a captured in-flight `Event` stays set, so a running compress is never un-cancelled. No permanently-uncompressable path remains.
- **`.rels` fail-closed core:** sentinel can't collide (matches no external prefix; `if target:` refuses any truthy); single consumer; entry-cap + scan-exception on a confirmed zip тЖТ fail closed; `ftp(s)` added; fail-open only for genuine non-zip OLE.
