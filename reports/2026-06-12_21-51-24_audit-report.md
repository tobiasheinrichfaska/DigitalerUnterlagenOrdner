**Architecture:** Logic/UI/infra separation is real and enforced — `core/` imports no `tkinter`/`webview` (grep-confirmed zero matches); Engine/render/CPU ports are DI'd and headless-testable; `host.py` is a thin per-window shell over a shared UI-free `CoreApi`. Cohesive and well-commented, so a quality note, not a defect.

## Verified SOUND (credited)

- **Bomb guards genuinely bound memory** — zip/tar do real incremental reads with a dedicated `_ArchiveTooLarge` re-raised past the `BadZipFile` catch (lying header can't bypass); variant blobs are double-capped (`variant_blobs.unpack` entry+read caps; `variant_store._read_attachment_capped` refuses non-`/FlateDecode`, inflates incrementally to `max+1` with a cross-attachment running total — fails closed *below* detonation).
- **`MAX_RENDER_PIXELS` applied on all raster paths**, including the compression path (`_render_one_page` → `_capped_dpi`), closing the old height-unbounded gap.
- **OOXML `.rels` pre-scan fails closed** (truncation/over-count/exception → `SCAN_UNREADABLE` → refusal); only non-ZIP legacy OLE stays fail-open (documented); hyperlinks correctly exempted.
- **Named-pipe IPC is properly restricted** — DACL grants only the current-user SID + LocalSystem; 256 MiB frame cap; JSON only, **no pickle** anywhere in load/IPC.
- **EXE/script magic-byte gate runs on both branches** (path + bytes/email-member); no archive re-amplification (no nested zip/tar dispatch).
- **No path traversal** — archive member names are display-only; extraction is fully in-memory; COM writes to fresh `mkdtemp`. HTML/email export blocks remote+local resources (`data:`/`cid:` only).
- **COM hardening** — `AutomationSecurity=3`, ReadOnly, UpdateLinks off, no recent-files, `Quit()` in `finally` for all three apps.
- **Replay invariant, `_after_mutate` cancel-token revive, identity-memoized version token, engine digest LRU+byte budget** — all correct and tested.
- **Frontend: no XSS surface** — no `dangerouslySetInnerHTML`/`innerHTML`/`eval`; all user/PDF-derived strings render as escaped React text; HelpModal external link uses `encodeURIComponent` + `rel="noreferrer"`; localStorage holds only UI prefs behind try/catch.
- **i18n claims hold** — `en.js` = exactly 151 keys (locked); key-set + placeholder parity tested across all 19 full languages (tlh/qya/sjn correctly exempted); en-US/en-GB are genuine thin overrides; no hardcoded user-facing strings outside `t()`.
- **lib/ logic** — selection resolve/`mergeableIds`, treeNav carry-move, tags filter/group (synthetic-id guards), status aggregation, messages reverse-templating — all correct, 74 lib test cases.

### Top fixes (Medium, high impact)
1. **F1 — gate global shortcuts off while modals are open** (`useKeyboard` `enabled`). Prevents `Delete`/Ctrl+S/arrows acting on the hidden tree behind a dialog. ~1 line.
2. **F2 — compute shift-range over `treeForView` in active filter views.** Stops silent status/export actions on hidden documents (a real silent-failure class).
3. **S1 — add member-count + total-size caps to email extraction**, matching the zip/tar guards. The one untrusted-import path left unbounded.

### Architectural improvements worth considering
1. **Extract collaborators out of `CoreApi`** (~1000 lines): a render/prefetch coordinator and a lock/save-pipeline object — the only real SRP stretch in an otherwise clean layering.
2. **Shared modal hook** for ExportDialog/SaveDialog/HelpModal (focus-first, Esc, focus-restore) — fixes F4 once and removes the per-dialog inconsistency; can fold in the deferred accessible-menu pattern (item 10).
