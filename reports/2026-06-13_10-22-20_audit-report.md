## Summary

**Totals:** 0 Critical · 0 High · 0 Medium · 2 Low.

**Top fixes (priority order):**
1. **L-1** — remove (or wire in) `get_filetypes_for_dialog()` + `is_supported()`; they are dead after
   the L-2 fix. One-line deletion is the safe default.
2. **L-2** — pin/comment the `DEV_URL` port so `BELEG_DEV` mode can't silently drift from `webui`'s
   dev server.

**No architectural improvements required** — the import subsystem (the focus of the last three passes)
is now coherent: single static `get_supported_extensions()` source of truth, both `convert()` branches
gated consistently, archive/e-mail caps symmetric, and `convert()` deliberately refuses to recurse into
nested archives (a member `.zip` falls through to the `else` at `importer.py:190-193` → "nicht
importierbar"), which is both the recursion-bomb guard and the (intended) no-nested-archive limitation.

**Quick wins:** both findings are < 5-line edits; L-1 is a pure deletion.

**Architectural note worth keeping (carried from prior pass, still open):** the unit suite cannot catch
"built-then-forgotten" wiring like L-1 because it tests the helpers in isolation. A tiny integration test
asserting `host._import_file_types()` actually surfaces Office/ODF extensions (and that no importer public
method is unreferenced) would have caught both the original L-1 and this residual. Optional.

**Already-tracked (not re-flagged):** the long-deferred error-code contract (CLAUDE.md Open item #11);
manual tests 01–04 describe legacy Tk flows (Known Limitation); the >100-page export auto-split not wired
into the UI (FEATURES_REQUIRED §5); the accepted audit-info residuals listed in CLAUDE.md (2026-06-12).
