| F-1 | Documentation | Low | Confirmed | `Node.to_dict()` emits a `has_source` UI-hint key that is **not** listed in the documented JSON/persisted shape in `docs/data-model.html`. All persisted fields (tags, collapsed, compression_no_gain, compression_method, vz_*, dpi_*) are covered; only this derived field is missing. | `core/model.py:82` (emitted) vs `docs/data-model.html` (absent) | Add `has_source` to the Node `to_dict` field table in the data-model doc, marked as a **derived** (non-persisted) UI hint = `original_data is not None`, so the doc fully mirrors the serialized shape. |

(HTML email bodies are unaffected — they route through `html_to_pdf`.) | `universal_importer/converters.py:65` | Soft-wrap long lines (e.g. break at the character budget and continue on the next line) rather than slicing, so no content is dropped. If truncation is intentional for a layout reason, document it as an accepted limitation. |

## Notes on things explicitly verified as NOT issues

- **`host.py` DEV_URL = `localhost:5173`** intentionally tracks Vite's default, **not** the workspace 5178 web-server port — the comment (host.py:71–74) documents this is an embedded-SPA dev convenience, and a prior audit already addressed the coupling. Not a finding.
- **Ruff absent** from `.build_venv` — that venv is the PyInstaller build environment (runtime + build deps), not a lint/dev env. Not a project code finding.
- **No XSS sinks** in the React UI (`dangerouslySetInnerHTML`/`innerHTML`/`eval`/`new Function` — zero matches).
- **No shell injection** — the only `subprocess` use is a test (`tests/test_file_lock.py`), list-form, no `shell=True`.
- **Nested-archive amplification is closed**: a `.zip`/`.tar` *inside* an archive is routed back through `UniversalImporter.convert`, where the path branch has no handler for archive extensions → it becomes "nicht importierbar" rather than recursively expanding (so each archive's 500 MB/500-member budget can't be multiplied).
- **vitest 4.1.8 forks-pool crash** (known on this machine) is already worked around with `pool:'vmThreads'` (latest commit f1e8a09); all 216 web tests run green.
