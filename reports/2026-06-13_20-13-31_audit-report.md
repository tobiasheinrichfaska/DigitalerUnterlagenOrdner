### F-2 fix note (deeper defect found & fixed)

The original code was `c.drawString(margin, y, line[:120])`. The first-pass fix (fixed
120-char chunks) **still failed** the regression test: at the default font, a 120-character
line is **wider than the A4 text column**, so reportlab draws the overflow off the page edge
where it is clipped and unrecoverable. In other words, the pre-fix behaviour lost content in
**two** ways — an explicit `[:120]` truncation *and* horizontal clipping of anything past
roughly column ~68. The shipped fix wraps by *measured* width, so both loss paths are closed:
the output may now span more lines/pages, but no characters are lost.
`converters.py:54` (`txt_to_pdf`).
