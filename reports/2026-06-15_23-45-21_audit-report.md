### Security review — no findings (verified strong)
- **Import gate** (`universal_importer/importer.py:53-70`): EXE/script signature refusal (MZ/ELF/Mach-O/shebang) + per-extension magic-byte check on **both** the path and bytes branches. Confirmed.
- **OOXML SSRF / NTLM-leak guard** (`converters.py:169-212`): `.rels` external-target pre-scan, fail-**closed** on unreadable/oversized/over-entry ZIPs (`SCAN_UNREADABLE`), runs before any COM open; ODF/legacy-OLE fail-open documented and covered by COM guards (`AutomationSecurity=3`, `ReadOnly`, `UpdateLinks` off). Confirmed.
- **HTML/email LFI + remote fetch** (`converters.py:98-114` `block_remote_link`): only `data:`/`cid:` inline resources allowed; `http(s)`/`file://`/relative blocked. Confirmed.
- **Zip/tar/email bomb guards** (`archives.py`): entry-count cap + declared-size pre-check + actual-read running-total cap (500 MB) across all container paths incl. email body/attachments. Confirmed.
- **Render allocation bombs** (`services/render.MAX_RENDER_PIXELS`): DPI clamp on preview and compression raster paths. Confirmed (per CLAUDE.md; consistent with code structure).
- **Variant blob bombs** (`variant_blobs.unpack` + `_read_attachment_capped`): dual-layer 500 MB / 500-entry caps, no uncapped `read_bytes()`. Per CLAUDE.md.
- **Path/filename safety** (`core/api.py:885-892` `materialize_subset`): illegal-char strip + Windows-reserved-name guard + trailing dot/space strip before path use. Confirmed.
- **No XSS**: no `dangerouslySetInnerHTML` / `innerHTML` / `eval` in `webui/src`. Confirmed.
- **PII discipline**: email parsing logs structure only, never subject/body content (`archives.py:199-221`). Confirmed.
- **Temp-file hygiene**: COM conversion uses `mkdtemp` + `finally` cleanup + Office `Quit()` in `finally` (`converters.py:240-317`); import-bytes uses `mkdtemp` + `rmtree` in `finally` (`api.py:987-994`). Confirmed.

### Architecture & structure — healthy
- Clean layering: pure data-driven `core/` (immutable `Node`/`Document`, frozen commands + reducer, session/undo) ↔ stateful `services/` ↔ `formats/` I/O ↔ `infra/` ports ↔ `host.py` (the single pywebview entry). UI-agnostic logic is fully unit-tested headless.
- Concurrency is carefully reasoned: a documented lock order (`CoreApi._lock → engine._dcache_lock → engine._mcache_lock`), per-session lock/cancel-token lifecycle, and the few deliberate unlocked accesses are annotated as benign (atomic tuple swaps). The accepted info-level residuals are listed in CLAUDE.md "Open / deferred".
- Error handling: broad `except` blocks are intentional (never crash the host / best-effort cleanup) and `logger.exception`-traced where a silent swallow would hide a leak (e.g. `host.py:111-119` close guard).
